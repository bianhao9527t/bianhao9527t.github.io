通信和网络

  1. 网络通信的例子
  • A.两台计算机通过一根网线相连，就组成了一个最简单的网络。
  • B.由一台路由器（或交换机组成的小型网络）
  • C.当计算机想从某个网址获取文件时，必须先接入internet然后才能下载

信息传递的过程

  • 类似与快递站 物品-包裹-集散中心-天空-集散中心-包裹-物品
  • 计算机发送报文-网关路由器-lnternet-网关路由器-计算机
  • 都是需要对方的地址

局域网，城域网，广域网

  • 网络类型可以根据覆盖的地理范围，划分为局域网和广域网，以及介于局域网和广域网之间的城域网
  • 局域网特点：覆盖范围几公里，分布近等 使用技术：以太网，wi-fi等。（手机热点也是局域网）
  • 城域网特点：范围大，成本高，速度快等 主要功能链接数据库及局域网等
  • 广域网跟城域网相似，但实现方式和性能不同 使用的技术：基于大型的局域网，与局域网技术相似，如：以太网（10gbps/100gbps）wimax（全球互通微波访问）

什么是IP地址

  • ip地址就像现实中的地址，可以标识网络的一个节点，数据就是通过它来找到目的地
  • ip地址是网络设备接口的属性，不是网络设备本身的属性。当我们说给某台设备分配一个ip地址时，实质上是指这台设备的某个接口分配一个ip地址。如果设备有多个接口，通常每个接口都至少需要一个ip地址
  • 注：需要使用ip地址的接口，通常是路由器和计算机的接口

ip地址的表现

  • ip地址表示：ip地址长度是32bit，由4个字节组成，为了方便阅读和书写，ip地址通常采用点分十进制数来表示
  • 点分十进制表示法，二进制和十进制
  • ipv4地址范围：0.0.0.0~255.255.255.255.

ipv4和ipv6的区别

  • ipv4协议族是tcp/ip协议族最为核心的协议族
  • ipv6是网络层协议的第二代标准协议.ipv4的升级版本

私有ip地址

  • 公网ip地址：连接到internet的网络设备必须具有ICANN分配的公网ip地址
  • 私网ip地址：私网ip地址的使用使得网络可以得到更为自由的扩展，因为同一个私网ip地址是可以在不同的私有网络中可以重复使用的
  • 私网与internet的互联，必须使用网络地址转换（NAT）技术实现
  • NAT，网络地址转换，其基本作用是实现私网ip地址和公网ip地址之间的转换
  • IANA,因特网地址分配组织

特殊ip地址

  • 有限广播地址：255.255.255.255（地址范围） 作用可作为目的地址，发往该网段所有主机（受限于网关）路由器接收到目的ip地址为有限广播地址的ip报文后，会停止对该ip的报文转发

  • 任意地址 0.0.0.0（地址范围）“任何网络”的网络地址；“这个网络上这个主机接口”的ip地址 当一个主机接口在启动过程中尚未获得自己的IP地址时，就可以向网络发送目标的ip地址为有限广播地址，源ip地址为0.0.0.0的DHCP请求报文，希望DHCP服务器在收到自己的请求后，能够给自己分配一个可用的ip地址

  • 环回地址 127.0.0.0/8 (地址范围)测试设备自身的软件系统 （/8 是指而二进制前八位为都是1）一个设备产生的，目的ip地址为环回地址的ip报文是不可能离开这个设备本身的

  • 本地链路地址169.254.0.0/24（地址范围）当主机自动获取地址失败后，可使用该网段中某个地址进行临时通信（24/ 是指十进制前24位都是1）如果一个网络设备获取ip地址的方式被设置成了自动获取方式，但是该设备在网络上又没有找到可用的DHCO服务器，那么该设备就会使用169.254.0.0/16网段的某个地址来进行临时通行通信（要是看到169开头的IP基本就gg了）

端口扫描

  • Web指纹定义 ：Web指纹是一种对目标网站的识别技术，通过识别网站所使用的服务器、运行的脚本、安装的CMS等信息对目标进行精准的分类和定位。Web指纹识别可以帮助安全研究人员在安全测试中快速了解目标网站的基本信息，有助于搜索其相关漏洞。
  • 指纹是指对目标网站的识别技术
  • 端口号有1~65535个
  • 桥接模式（将虚拟机直接连接到外部网络 也就是跟主机平起平坐）
  • NAT模式（与虚拟机共享主机的ip地址 相当于公网中的私网）

利用github搭建hexo博客

准备前提

  1. 提前安装好(node.js)[https://nodejs.org/en] 左边的那个
  2. 提前安装好(GIt)[https://git-scm.com/downloads]
  3. 提前注册好一个github账号
  4. 如果文章图片没有显示，请自行挂代理

操作步骤

  1. 在本地找个目录放博客源文件，路径不要有中文
  2. 在选定的目录中，右键Git Bash Here
  3. 使用npm命令安装Hexo，输入：

npm install -g hexo-cli //安装hexo
hexo init blog //初始化一个名为blog的项目
cd blog //进入blog
hexo g //编译构建

构建好的效果

cd进入这个目录
为了检测我们网站的雏形，分别按顺序输入以下三条命令：
hexo new test_my_site //博客新建一个文章
hexo g //构建
hexo s //启动服务
之后就可以访问127.0.0.1:4000访问blog了

cd~/.ssh#检查本机已存在的ssh密匙

  • key填写C:\Users\Administrator.ssh\id_rsa.pub中的内容
  • 尝试ssh连接,测试是否成功
  • ssh -T git@github.com # 注意邮箱地址不用改

hexo命令介绍
hexo n “我的博客” == hexo new “我的博客” #新建文章
hexo new page “pageName” #新建页面
hexo g == hexo generate #生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy #部署
hexo server #Hexo会监视文件变动并自动更新，无须重启服务器
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP
hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令
hexo version #查看Hexo的版本
hexo generate #生成静态页面至public目录
hexo s -g #生成并本地预览
hexo d -g #生成并上传

hexo目录结构
+– .deploy      #hexo deploy生成的文件
+– node_modules  #npm组件
+– public        #生成的静态网页文件
+–scaffolds      #模板
+– source        #博客正文和其他源文件
|  +– _posts    #我们自己写的文章以md结尾
+– themes        #主题
+– _config.yml  #全局配置文件
– package.json  #定义了hexo所需要的各种模块
